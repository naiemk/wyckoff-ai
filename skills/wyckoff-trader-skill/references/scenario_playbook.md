# Scenario Playbook

Use this file when the task is to turn observations into a concrete Wyckoff scenario.

## Goal

Produce a scenario that is:

- structurally grounded
- explicit about uncertainty
- usable for trade planning or no-trade decisions
- adapted to crypto market internals when relevant

## Scenario Checklist

### 1. Context

State:

- asset and timeframe
- higher-timeframe cycle position
- buying / selling / neutral position
- intermarket backdrop if relevant

### 2. Wyckoff Story

Describe the structure in plain language before labels:

- where did stopping action appear?
- where did supply or demand get absorbed?
- what level keeps getting tested?
- what says the path of least resistance is up, down, or still unclear?

### 3. Phase And Event Evidence

List the evidence behind the current read:

- probable phase
- candidate events
- strongest supporting observations
- contradictory observations

If evidence is weak, say the read is provisional.

### 4. Crypto Overlay

Check:

- S&P / Nasdaq / dollar dependency
- Bitcoin leadership or weakness
- cap-tier rotation
- thematic-sector strength
- relative strength versus benchmark

If the task includes choosing among multiple candidates, use a top-down drilldown:

- broad market
- sector or theme
- industry group or crypto bucket
- best individual candidate

### 5. Leading Scenario

Include:

- thesis
- trigger
- confirmation behavior
- invalidation
- likely path
- confidence level in words, not fake precision

### 6. Alternate Scenario

At minimum include one alternate that could realistically happen if the trigger fails or intermarket changes.

### 7. Tactical Quality

Classify the setup:

- `Phase C aggressive`
- `Phase D confirmation`
- `Phase E continuation`
- `watchlist only`
- `no-trade`

### 8. What Would Change The Read

Name the behaviors that would force reassessment:

- failed signal
- unexpected effort/result divergence
- loss of relative strength
- intermarket break
- inability to hold a test

## Recommended Output Template

```text
Context
- ...

Wyckoff Story
- ...

Phase / Event Read
- ...

Crypto Overlay
- ...

Leading Scenario
- Trigger:
- Invalidation:
- Expected path:

Alternate Scenario
- Trigger:
- Invalidation:
- Expected path:

Decision
- Trade / Wait / No-trade:
- Why:
```

## Ranking Heuristics

Prefer scenarios with:

- clear higher-timeframe context
- coherent effort/result behavior
- a tested signal, not just an initial signal
- supportive relative strength
- supportive intermarket backdrop
- defined invalidation
- constructive trend stride when already in markup or markdown
- leadership that is `in gear` versus relevant peers or benchmarks

Downgrade scenarios with:

- mixed phase evidence
- weak or absent test
- lagging relative strength
- obvious dependence on fragile macro conditions
- narrative built mostly from labeling

## What To Avoid

- Do not treat every range as accumulation or distribution.
- Do not confuse relative strength with automatic long bias.
- Do not issue a directional scenario without naming the invalidation level or behavior.
- Do not force a spring or UTAD label if the behavior fits an LPS/LPSY-led variant better.
